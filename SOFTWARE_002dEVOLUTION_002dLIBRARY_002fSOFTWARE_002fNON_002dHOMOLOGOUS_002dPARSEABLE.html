A.45 SOFTWARE-EVOLUTION-LIBRARY/SOFTWARE/NON-HOMOLOGOUS-PARSEABLE

Generic Function: ast-annotation-to-rebind [sel/sw/non-homologous-parseable] ast

Return the AST annotation field to be rebound as part of variable rebinding.

Generic Function: ast-type-to-rebind-p [sel/sw/non-homologous-parseable] ast

Return t if AST is of a type where its variables/functions should be rebound.

Generic Function: check-interleaved-text [sel/sw/non-homologous-parseable] ast

Assert that AST’s interleaved text is valid.

Interleaved text is valid when the number of interleaved strings is one greater than the number of children.

Generic Function: interleaved-text [sel/sw/non-homologous-parseable] obj

Get the interleaved text for obj.

Method: interleaved-text [sel/sw/non-homologous-parseable] (interleaved-text interleaved-text)

Interleaved text between children.

Function: children-definitions-table [sel/sw/non-homologous-parseable] children-definitions

Build a hash table children-definitions, an alist from a list of keys to a list of (child . arity).

The keys of the hash table are the individual keys of each entry.

Function: convert-helper [sel/sw/non-homologous-parseable] spec prefix superclass children-definitions

Common function for converting an AST SPECification list into an AST.

  • spec AST specification association list
  • prefix prefix for type name and slots
  • superclass superclass of the AST type to create
  • children-definitions hash table mapping AST types to their child slots
Function: expand-ast-classes [sel/sw/non-homologous-parseable] superclass prefix spec

Returns a list of AST node definitions derived from spec with the given superclass and prefix.
