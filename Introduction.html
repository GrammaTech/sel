1 Introduction

The Software Evolution library enables the programmatic modification and evaluation of extant software.

A common interface abstracts over multiple types of software objects (see Software Objects) including abstract syntax trees parsed from source code, LLVM IR, compiled assembler, and linked ELF binaries. Mutation and evaluation methods (see Software Methods) are implemented on top of this interface supporting automation of common software engineering tasks. Support is included for Search Based Software Engineering (SBSE) techniques (see Search Functions).

1.1 SEL Applications

SEL has a broad range of applications for software improvement and understanding. The following are some current and past projects utilizing SEL.

  • Multiple functions underlying an integrated software development assistant. (https://grammatech.gitlab.io/Mnemosyne/docs/)
  • Automatically resolving merge conflicts (https://mergeresolver.github.io).
  • Optimizing real-world software (github.com/eschulte/goa).
  • Patching vulnerabilities in closed source binaries (eschulte.github.io/netgear-repair).
  • Filling "holes" in incomplete programs whose behavior is partially specified by unit tests and developer hints (MuSynth).
  • Injecting bugs into software to create cyber-defense benchmarks (https://arxiv.org/pdf/1901.02819.pdf).
  • Decompiling binaries to C source ( BED (Byte Equivalent Decompilation)).
  • Studying the mutational robustness of software (cs.unm.edu/~eschulte/dissertation).

1.3 Other features

SEL provides support for exploring many different aspects of both evolutionary search and automated software manipulation.

  • Multiple search techniques including steady-state and generational evolution or Markov chain Monte Carlo (MCMC).
  • Custom mutations tailored to each software object.
  • One- and two-point crossover.
  • Lexicase or multi-objective fitness evaluations.
  • Use of "fodder databases" as a source of new statements or expressions that can be injected by mutations.
  • Ancestry tracking of evolved software objects.
  • Adaptive mutations which dynamically update probabilities based on mutation success.
  • Super-mutants to more efficiently evaluate multiple mutations in a single genome.

Tailored features for Clang C and C++ objects

  • Automatic repair for some compilation errors and warnings.
  • Special mutations for renaming variables, changing loops, and otherwise manipulating statements.
