E.10 Don’t use superfluous let or let* bindings

Any let* which can be changed to a let should be changed to a let.

Typically, if a let-bound variable is only used once it should not be bound but instead its definition should replace its sole use. This is suggested because each variable binding forces every subsequent reader to perform a dereference. For example, this 

(let* ((subject (quick brown fox))
       (object (lazy dog))
       (sentence (The subject jumps over the object.)))
  sentence)

is harder to read than this. 

(The quick brown fox jumps over the lazy dog)

There are some exceptions to this rule.

  • If the variable is often used in debugging (printf or debugger) and it thus matters that it is bound at that point in the code then this may be acceptable.
  • If the let-bound value appears as, lets say, the third argument to some function the in-lining of the long calculation may obscure the flow of the function with it’s other arguments.
  • If use of the let avoids horrible indentation issues then it may be acceptable.
